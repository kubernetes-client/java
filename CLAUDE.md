# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This is the official **Kubernetes Java Client** (`io.kubernetes:client-java`) — a multi-module Maven project that generates and maintains Java bindings for the Kubernetes API. Current version: `27.0.0-SNAPSHOT`.

The `kubernetes/` module (this subdirectory) contains auto-generated API model and client code from `swagger.json`. **Do not hand-edit files in `kubernetes/src/` directly** — they are regenerated by `scripts/update-client.sh` using the OpenAPI generator.

## Module Structure

| Module | Artifact | Purpose |
|--------|----------|---------|
| `kubernetes/` | `client-java-api` | Auto-generated API models and typed API clients (CoreV1Api, AppsV1Api, etc.) |
| `util/` | `client-java` | Core utilities: `ClientBuilder`, `KubeConfig`, `Watch`, `Yaml`, `Exec`, `PortForward`, informers |
| `extended/` | `client-java-extended` | Higher-level abstractions: controllers, leader election, work queues, pagers, kubectl-style ops |
| `spring/` | `client-java-spring-integration` | Spring Boot auto-configuration and `SharedInformerFactory` integration |
| `spring-aot/` | `client-java-spring-aot-integration` | GraalVM native-image hints for Spring AOT |
| `fluent/` | `client-java-fluent` | Fluent builder DSL for Kubernetes resources |
| `proto/` | `client-java-proto` | Protobuf-encoded API for high-performance use cases |
| `client-java-contrib/` | — | Separately versioned CRD clients: cert-manager, prometheus-operator, admissionreview |
| `examples/` | — | Not released; smoke-tested in CI |
| `e2e/` | — | End-to-end tests, run against a real cluster (Kind); skipped by default |

## Build and Test Commands

All commands use the Maven wrapper (`./mvnw` / `mvnw.cmd`) from the repo root.

```bash
# Build and test all modules (excludes spring, spring-aot, e2e)
./mvnw clean test

# Build without tests (needed before running e2e)
./mvnw clean install -Dmaven.test.skip=true

# Test a specific module
./mvnw -pl util clean test
./mvnw -pl extended clean test

# Run a single test class
./mvnw -pl util -Dtest=ClientBuilderTest test

# Check formatting (Google Java Format via Spotless)
./mvnw spotless:check

# Apply formatting
./mvnw spotless:apply

# Spring module tests (uses Bazel, not plain Maven)
bazel test //spring:tests

# E2E tests (requires a running cluster, e.g. Kind)
./mvnw clean install -Dmaven.test.skip=true
cd e2e && ../mvnw test -De2e.skip=false
```

## Code Generation

The `kubernetes/` module is **fully generated**. To regenerate after an API spec update:

```bash
# Regenerate kubernetes/ from swagger.json using the gen repo
./scripts/update-client.sh

# After updating a managed dependency version in the root pom.xml,
# also update Bazel:
python3 scripts/sync_bazel_dependencies.py
REPIN=1 bazel run @maven//:pin
# Then commit MODULE.bazel and maven_install.json
```

## Architecture

### Client construction
`ClientBuilder` (in `util/`) is the entry point for creating `ApiClient` instances. It supports in-cluster service account credentials, kubeconfig files, exec-based credentials (e.g. EKS), and token files. `ClientBuilder.standard().build()` auto-detects the environment.

### Authentication
Credentials implement `Authentication` in `util/src/main/java/io/kubernetes/client/util/credentials/`. Key implementations: `KubeconfigAuthentication`, `TokenFileAuthentication`, `EKSAuthentication`, `ClientCertificateAuthentication`.

### Informers and controllers
`SharedInformerFactory` (in `util/`) manages `SharedIndexInformer` instances that watch Kubernetes resources. The `extended/` module builds on these with a controller reconciliation loop pattern: `Controller` → `Reconciler` interface, backed by a `WorkQueue`.

### Spring integration
The `spring/` module provides `@EnableKubernetesInformer` and auto-wires `SharedInformerFactory` into the Spring context. It uses Bazel (not plain Maven) for its test suite due to Bazel's better support for the multi-module dependency graph.

## Code Style

- Java 11 minimum, tested on 11/17/21.
- **Google Java Format** (v1.17.0) enforced via Spotless — run `./mvnw spotless:apply` before committing.
- No wildcard imports (`import foo.bar.*` is replaced with `INVALID IMPORTS` by Spotless).
- No Guava imports in non-generated code.
- Apache License 2.0 header required on all `.java` files.

## Bazel

Bazel is used alongside Maven, primarily for the Spring module CI and for contributors who prefer it. `MODULE.bazel` and `maven_install.json` are generated/updated by scripts — do not edit the `BEGIN/END generated` block in `MODULE.bazel` by hand.

## Key Concepts (Glossary)

| Term | Definition |
|------|------------|
| `ApiClient` | HTTP client wrapper (OkHttp3) holding base URL, auth, and TLS config; passed to every generated `*Api` class. |
| `ClientBuilder` | Fluent factory in `util/` for constructing `ApiClient`; auto-detects in-cluster vs kubeconfig vs exec credentials. |
| `KubeConfig` | Parses `~/.kube/config`; supports multiple clusters/contexts/users and exec-based credential plugins. |
| `Watch<T>` | Server-sent event stream over a list API call; yields `ADDED`/`MODIFIED`/`DELETED` events for a resource type. |
| `SharedIndexInformer<T>` | Resync-capable watch loop that maintains a local indexed cache; foundation for all controllers. |
| `SharedInformerFactory` | Manages lifecycle (start/stop/wait) of multiple `SharedIndexInformer` instances. |
| `Reconciler` | Interface in `extended/` implemented by controllers; `reconcile(Request)` is called for each queued object key. |
| `WorkQueue<T>` | Rate-limited, de-duplicating queue used to decouple watch events from reconciler execution. |
| `fluent builder` | `fluent/` DSL that mirrors the Kubernetes object model; e.g. `new V1PodBuilder().withNewMetadata()...`. |
| CRD model | Custom Resource Definition Java model, generated from a CRD YAML via the tooling in `client-java-contrib/`. |

## Constraints

**Never edit these artefacts by hand — they are fully generated:**

- `kubernetes/src/` — regenerated by `scripts/update-client.sh` from `swagger.json`. Any manual change will be lost on the next codegen run.
- The `# BEGIN … # END generated` block in `MODULE.bazel` — owned by `scripts/sync_bazel_dependencies.py`.
- `maven_install.json` — owned by `REPIN=1 bazel run @maven//:pin`.

**Handle with extra care (public API):**

- `util/` — `ClientBuilder`, `KubeConfig`, `Watch`, `Yaml`: widely used by downstream consumers; breaking changes require a major version bump.
- `extended/` — `Reconciler`, `Controller`, `WorkQueue`, `SharedInformerFactory`: same stability contract.
- `spring/` — any change to auto-configuration keys or bean names is a breaking change for Spring Boot users.

Do not add Guava or wildcard imports anywhere outside `kubernetes/src/`.

## Working With This Repo (Agent Rules)

**Ask a clarifying question before proceeding when:**
- The task touches public API signatures in `util/`, `extended/`, or `spring/`.
- The task involves regenerating or patching `kubernetes/src/` (clarify: is a full codegen run intended?).
- A dependency version change is requested (Bazel sync may also be required).

**Proceed with reasonable assumptions when:**
- Changes are confined to `util/` or `extended/` internals and are fully covered by existing or new unit tests.
- The task is a bug fix with no public API surface change.
- The task adds a new test class without touching production code.

**Required response format (always in this order):**
1. **Plan** — what will change and why, modules affected.
2. **Code changes** — minimal diff; new `.java` files must carry the AI compliance banner (see below).
3. **Rationale** — non-obvious design decisions only.
4. **Verification steps** — concrete commands:
   ```bash
   ./mvnw -pl <module> test          # targeted tests
   ./mvnw spotless:check             # formatting
   ./mvnw -pl <module> clean test    # full module verification
   ```

## Common Task Playbooks

### Add a utility method to `util/` with a test
- **Prompt example:** "Add a `Strings.truncate(String, int)` helper and test it."
- **Expected format:** Plan → new method in `util/src/main/java/…/util/Strings.java` → new `@Test` in `util/src/test/…/StringsTest.java` → Rationale → Verification.
- **Definition of Done:** `./mvnw -pl util test` passes; `./mvnw spotless:check` passes; no public API changed.

### Cover an existing class with tests
- **Prompt example:** "Add unit tests for `WaitUtils`."
- **Expected format:** Plan → new test file only → Verification.
- **Definition of Done:** `./mvnw -pl util test` passes; coverage increase visible; no production code changed.

### Fix a bug without changing public API
- **Prompt example:** "Fix NPE in `KubectlTop` when pods are down (see issue #4087)."
- **Expected format:** Plan (confirm no API signature change) → minimal production code fix → regression test → Verification.
- **Definition of Done:** `./mvnw -pl extended test` passes; `./mvnw spotless:check` passes; the offending scenario is covered by a new or updated test.
